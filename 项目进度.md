# Scales & Chroma 项目进度

更新时间：2026-05-30

## 当前项目范围约定

- 成长系统继续保留，但后续复杂优化优先级降低，先保证主流程完整。
- 认证与用户体系只做最基础版本：
- 注册不需要验证码。
- 密码阶段性不做加密存储。
- 核心目标是保证每个用户只能访问自己的练习、组合和成长数据。
- 正式上线阶段默认使用 `IP + 端口` 访问，不以域名部署为当前目标。
- 数据库继续走轻量化路线，以 SQLite 为最终目标方案。
- 错误监控保持基础能力即可，不追求复杂平台化接入。
- API 边界处理会明确写进测试规划，方便后续答辩展示测试设计思路。

## 当前已经完成的核心能力

### 1. 乐理视觉沙盘

- 前端已经有可用的 React + TypeScript + Vite 沙盘页面。
- 乐理元素库已覆盖音阶、调式、和弦、和弦进行。
- 支持拖拽、编排、替换、删除乐理积木。
- 已优化拖拽工作台：中间舞台与底部轨道保持在视口内，舞台本身也能直接接住拖拽，解决“轨道被挤到很下面、不容易放置”的问题。
- Canvas 实时渲染已接入视觉参数，可以响应颜色、辉光、粒子、几何与动画状态。
- 后端 `/sandbox/render` 可以根据乐理组合返回视觉参数，并结合用户已解锁特效增强渲染。

### 2. 视觉引擎

- 后端已有独立 `visual_engine` 模块，包含色彩映射、张力分析、粒子、几何和动画控制。
- 前端已有 `visual_engine/mapTheoryToVisuals.ts` 作为本地兜底映射。
- 视觉引擎已重构为“参数化映射系统”：
- 每个乐理模块不再只给一个颜色，而是同时决定主色、辅色、背景色、张力、复杂度、运动速度、波纹、光束、颗粒密度、几何权重等参数。
- 当前又进一步补上了 `temperature / symmetry / depth / pulse density` 这组舞台级参数，让不同模块组合不只是在“换色”，而是在重写舞台的空间感、镜像感和节奏密度。
- 多个模块组合时会进行参数叠加，而不是简单覆盖。
- 特定合理组合会触发额外加成，例如 `Lydian + Maj7`、`Harmonic Minor + Dim7`、`II-V-I + Maj7`。
- 组合签名库已继续扩充，例如 `Melodic Minor + Dominant7`、`Mixolydian + Dominant7`、`Dorian + II-V-I`、`Ionian + I-V-vi-IV`。
- Canvas 舞台已升级为多层背景、动态 halo、环形波纹、光束场、碎裂/波纹/晶格/软球体几何、粒子轨迹与颗粒纹理的综合演出。
- Canvas 舞台又新增了“舞台建筑层”和“脉冲星群层”，现在不同签名会出现更明显的拱形天幕、地平线晶格、镜像地板、碎片风暴、棱镜幕帘等风格差异。
- Growth 解锁现在不只是“数值略增”，而是会直接把舞台拉向不同风格家族，例如：
- Neo Soul 更偏 `Velvet Tide`
- Metal 更偏 `Metal Shrapnel`
- Jazz 更偏 `Jazz Skyline`
- Fusion 更偏 `Fusion Prism`
- 五声音阶成长更偏 `Neon Trail`

### 3. 练习与成长闭环

- 支持创建练习记录：日期、时长、BPM、主题、备注。
- EXP 系统已实现 BPM 权重、连续签到加成、等级计算。
- 支持最近练习历史列表展示，并已接入主题/日期范围筛选。
- 支持年度热力图数据接口和前端展示。
- 支持技能树状态接口和前端展示，节点等级按累计练习分钟数增长。
- 支持五声音阶、Jazz II-V-I、Metal Sweep、Neo Soul Maj7 等主题解锁视觉能力。
- 已补上 Fusion 方向的视觉解锁规则。
- 前端已经展示永久解锁的视觉特效列表，让用户看到成长对沙盘能力的影响。
- 右侧已经新增 bonus 说明文案和中文特效说明，用户能直接读懂当前舞台是如何被组合和成长共同塑形的。

### 4. 最基础认证与用户隔离

- 已支持注册、登录、退出和会话恢复。
- 前端会把 token 保存在本地，并在刷新后恢复当前登录状态。
- 后端 `practice-records`、`heatmap`、`skill-tree`、`unlocked-effects`、`sandbox/render`、`compositions` 已全部改为按当前登录用户隔离数据。
- 不再依赖前端传 `user_id` 决定读取谁的数据，避免最基础的越权问题。
- 密码目前按项目约定保持明文方案，先满足轻量化上线和用户隔离目标。
- 后端 CORS 已支持 `localhost` 和 `IP + 端口` 形式的前端访问，便于后续直接部署到服务器用 IP 调试。

### 5. 基础上线能力

- 后端已支持直接托管 `frontend/dist`，项目可以改为单进程、单端口运行。
- 前端在没有显式 `VITE_API_BASE_URL` 时，会在非 Vite 开发端口下自动使用同源后端。
- 新增 SQLite 轻量自迁移逻辑，旧库缺少 `users.password` 列时会自动补齐，不需要手动删库。
- 已补充 `scripts/run_single_port.sh`，可以一条命令构建前端并启动后端。
- 这意味着当前项目已经具备“构建前端 -> 启动后端 -> 用服务器 IP + 端口访问”的最基础上线路径。

### 6. 沙盘组合保存

- 支持读取并加载已保存组合。

## 当前主要接口

- `POST /auth/register`：注册并返回 token。
- `POST /auth/login`：登录并返回 token。
- `GET /auth/me`：读取当前用户信息。
- `POST /auth/logout`：注销当前 token。
- `POST /practice-records`：创建练习记录并返回 EXP、等级、连续签到、解锁特效。
- `GET /practice-records`：读取用户最近练习记录，支持主题和日期范围筛选。
- `GET /heatmap/yearly`：读取年度练习热力图。
- `GET /skill-tree`：读取技能树进度。
- `GET /unlocked-effects`：读取用户已经永久解锁的视觉能力。
- `POST /sandbox/render`：根据乐理组合和用户解锁状态返回视觉参数。
- `POST /compositions`：保存沙盘组合。
- `GET /compositions`：读取用户保存的沙盘组合。
