# Scales & Chroma 项目进度

更新时间：2026-05-30

## 当前项目范围约定

- 成长系统继续保留，但后续复杂优化优先级降低，先保证主流程完整。
- 认证与用户体系只做最基础版本：
- 注册不需要验证码。
- 密码阶段性不做加密存储。
- 核心目标是保证每个用户只能访问自己的练习、组合和成长数据。
- 正式上线阶段默认使用 `IP + 端口` 访问，不以域名部署为当前目标。
- 数据库继续走轻量化路线，以 SQLite 为最终目标方案。
- 错误监控保持基础能力即可，不追求复杂平台化接入。
- API 边界处理会明确写进测试规划，方便后续答辩展示测试设计思路。

## 当前已经完成的核心能力

### 1. 乐理视觉沙盘

- 前端已经有可用的 React + TypeScript + Vite 沙盘页面。
- 乐理元素库已覆盖音阶、调式、和弦、和弦进行。
- 支持拖拽、编排、替换、删除乐理积木。
- 已优化拖拽工作台：中间舞台与底部轨道保持在视口内，舞台本身也能直接接住拖拽，解决“轨道被挤到很下面、不容易放置”的问题。
- Canvas 实时渲染已接入视觉参数，可以响应颜色、辉光、粒子、几何与动画状态。
- 后端 `/sandbox/render` 可以根据乐理组合返回视觉参数，并结合用户已解锁特效增强渲染。

### 2. 视觉引擎

- 后端已有独立 `visual_engine` 模块，包含色彩映射、张力分析、粒子、几何和动画控制。
- 前端已有 `visual_engine/mapTheoryToVisuals.ts` 作为本地兜底映射。
- 视觉引擎已重构为“参数化映射系统”：
- 每个乐理模块不再只给一个颜色，而是同时决定主色、辅色、背景色、张力、复杂度、运动速度、波纹、光束、颗粒密度、几何权重等参数。
- 当前又进一步补上了 `temperature / symmetry / depth / pulse density` 这组舞台级参数，让不同模块组合不只是在“换色”，而是在重写舞台的空间感、镜像感和节奏密度。
- 视觉引擎这轮又补上了 `valence / arousal / luminosity / grit` 四条情绪轴，用来描述“明暗情绪、唤醒度、发光空间、粗粝颗粒感”，让组合先形成情绪骨架，再喂给舞台。
- 多个模块组合时会进行参数叠加，而不是简单覆盖。
- 特定合理组合会触发额外加成，例如 `Lydian + Maj7`、`Harmonic Minor + Dim7`、`II-V-I + Maj7`。
- 组合签名库已继续扩充，例如 `Melodic Minor + Dominant7`、`Mixolydian + Dominant7`、`Dorian + II-V-I`、`Ionian + I-V-vi-IV`，以及 `Skyline Halo`、`Glass Current`、`Neon Lantern`、`Liquid Aurora`、`Copper Skyline` 等新签名。
- Canvas 舞台已升级为多层背景、动态 halo、环形波纹、光束场、碎裂/波纹/晶格/软球体几何、粒子轨迹与颗粒纹理的综合演出。
- Canvas 舞台又新增了“舞台建筑层”和“脉冲星群层”，现在不同签名会出现更明显的拱形天幕、地平线晶格、镜像地板、碎片风暴、棱镜幕帘等风格差异。
- Canvas 舞台进一步加入了情绪轴驱动的氛围层，不同组合现在会出现更明显的高空光幕、粗粝地表雾带、光晕扩展和高唤醒光束密度差异。
- Growth 解锁现在不只是“数值略增”，而是会直接把舞台拉向不同风格家族，例如：
- Neo Soul 更偏 `Velvet Tide`
- Metal 更偏 `Metal Shrapnel`
- Jazz 更偏 `Jazz Skyline`
- Fusion 更偏 `Fusion Prism`
- 五声音阶成长更偏 `Neon Trail`
- 当前舞台家族已经明确区分为 `日光穹庭 / 丝绒厅堂 / 金属熔炉 / 和声教堂 / 棱镜阵列 / 夜潮剧场 / 霓虹网格 / 影纹祭坛`，Growth 与组合会共同把舞台推向其中之一。
- 右侧已经加入 `Mood Axes` 面板，会把 `Valence / Arousal / Luminosity / Grit` 直接画成条带，方便第一眼比较不同舞台人格。
- 这一轮又把 Growth 单独拆成了 `Growth Imprint` 层：
- 现在不是只有“场景家族 + 组合签名”在变化，成长方向也会独立产出第二层舞台人格。
- 当前支持 `Pentatonic Drive / Jazz Lattice / Metal Forge / Neo Soul 幕纱 / Fusion Phase` 五种成长印记。
- 即使同一组模块相同，不同成长印记也会额外覆盖出霓虹速度线、和声吊灯、熔炉碎片雨、丝绒幕纱、相位棱镜这些明显不同的演出层。
- 这轮还补上了 4 条更贴近音乐结构的特征轴：
- `Openness`：开放度，决定扇面、展开感和空间张开程度。
- `Attack`：起音攻击性，决定尖峰、瞬时切面和刺点。
- `Swing`：摆动感，决定舞台里横向摆移和不完全平均的流动。
- `Gravity`：终止牵引力，决定舞台是否有明显“往中心或地平线收束”的下拉感。
- 这些特征会和情绪轴、Growth 印记一起叠加，形成第三层更像乐理而不是纯视觉的参数语言。
- 这一轮继续把组合逻辑从“命名组合库”扩成“系统性协同层”：
- `Resonance`：模块之间是否互相放大、彼此顺手。
- `Cadence Pull`：进行和和弦是否形成明显终止牵引。
- `Modal Tension`：调式与和弦之间是否存在高摩擦和阴影拉扯。
- `Blend Cohesion`：不同类型模块是否融合成统一色彩和空间结构。
- 当前已经能触发 `Radiant Voicing / Cadential Lift / Groove Pocket / Shadow Magnet / Color Convergence` 这类协同标签。

- 支持最近练习历史列表展示，并已接入主题/日期范围筛选。
- 支持年度热力图数据接口和前端展示。
- 支持技能树状态接口和前端展示，节点等级按累计练习分钟数增长。
- 支持五声音阶、Jazz II-V-I、Metal Sweep、Neo Soul Maj7 等主题解锁视觉能力。
- 已补上 Fusion 方向的视觉解锁规则。
- 前端已经展示永久解锁的视觉特效列表，让用户看到成长对沙盘能力的影响。
- 右侧已经新增 bonus 说明文案和中文特效说明，用户能直接读懂当前舞台是如何被组合和成长共同塑形的。
- 右侧又新增 `Stage Reading`，会把当前舞台的情绪、空间、运动和主要来源直接翻译成中文解读，降低首次使用门槛。
- `Stage Reading` 现在已被上移到右侧更靠前的位置，避免首次打开时先被一串参数读数盖过去。
- 右侧还新增了 `Growth Imprint` 面板，会直接告诉用户当前成长风格是哪一种、强度多少，以及它在舞台里具体改写了哪些部分。
- 右侧新增 `Harmonic Traits` 面板，会把 `Openness / Attack / Swing / Gravity` 用条带直接展示出来。
- 右侧新增 `Theory Synergy` 面板，会把协同轴和协同标签直接展示出来。
- 这一轮继续把“三元组合”的舞台升级成了 `Scene Cascade` 系统：
- 现在不只是两两组合命名，而是当 `Lydian + Maj7 + II-V-I`、`Dorian + Min7 + II-V-I`、`Harmonic Minor + Dim7 + Dominant7`、`Melodic Minor + Dominant7 + Aug`、`Pentatonic + Mixolydian + Dominant7` 这类三元结构成立时，会额外触发大型场景级联。
- 当前级联家族包含 `Aurora Dais / Velvet Arcade / Forge Ritual / Prism Vortex / Tide Runway / Eclipse Altar`。
- 这些级联不是只在右侧显示名字，而是会直接给 Canvas 追加大型拱幕、长廊、锻造架、棱镜旋涡、地平跑道、祭坛环等第二层舞台结构。
- 后端 `/sandbox/render`、前端本地映射、Canvas 渲染器和右侧解释面板都已经接入 `scene_cascade / scene_cascade_intensity`。
- 这轮又把前端渲染器改成“同一舞台连续变形”模式：
- 不再因为视觉状态变化而重建整个 Canvas 渲染器。
- 现在组合变化、Growth 解锁、场景级联触发时，会以平滑 morph + 冲击波过渡的方式切场，而不是瞬间跳变。
- 这让用户更容易直接感知“刚刚到底发生了什么变化”，也让三元组合和成长解锁的舞台升级更像演出而不是参数刷新。
- 这轮还补上了“Growth 改写级联”的分叉：
- 同一个三元组合，不同 Growth 轨迹现在会把同一个 `Scene Cascade` 往不同人格继续推深。
- 例如明亮级联会被 `Jazz Lattice` 推成更像和声窗格的 `Choir Vault`，也会被 `Neo Soul 幕纱` 推成更柔性的 `Silken Halo`。
- 暗色祭坛级联会被 `Metal Forge` 压成更硬的 `Forge Throne`，棱镜旋涡级联会被 `Fusion Phase` 继续扭成 `Phase Cloister`。
- 这类分叉已经接入后端规则、前端 Canvas 和右侧 `Scene Cascade` 解读，所以现在同一组合在不同成长风格下不再只是“多一层粒子”，而是连级联气质都会变化。
- 这轮又补了 `Growth Lens Preview`：
- 用户现在可以直接在右侧切到 `真实成长 / Jazz / Neo Soul / Metal / Fusion / Pentatonic` 这些预览镜头。
- 这个预览不会写入数据库，也不会伪造真实解锁数据，只会把当前组合临时送进 `/sandbox/render` 的 `preview_growth_imprint` 路径里。
- 这样同一组乐理积木可以被立刻拿来比较不同成长路线下的舞台差异，更适合演示、调参和答辩讲解。
- 这轮继续把大型舞台人格做成了独立 `Stage Setpiece` 层：
- 当 `Choir Vault / Silken Halo / Rose Arcade / Blue Cloister / Forge Throne / Phase Cloister / Neon Causeway` 这类成长分叉命中时，不再只是 `active_bonuses` 多出一个词。

- 右侧也新增了 `Stage Setpiece` 解读面板，会直接说明当前舞台被哪套大型装置接管、骨架是什么、体感为什么变了。

- 右侧 `Stage Setpiece` 面板也新增了“灯光调度”解释，现在用户能更直接看懂为什么两个大型舞台会有完全不同的现场气质。
- 这轮又继续补了一层 `Stage Director Cue`：

- 这层也已经接入中间舞台、左上角角标和右侧 `Stage Projection Script` 面板，所以不同 Growth 分叉现在连脚下的空间语法都彻底分开了。
- 这轮继续把近景机械和幕纱动作拆成了 `Stage Motion Rig`：
- 它专门负责“镜头前方到底有什么机构在动、它是往下压、往前冲、还是在近景包裹视口”。
- 现在 `Choir Vault` 会走 `Choir Crowns`，`Silken Halo` 会走 `Ribbon Veils`，`Forge Throne` 会走 `Piston Frames`，`Phase Cloister` 会走 `Prism Gates`，`Neon Causeway` 会走 `Runway Drones`，`Eclipse Altar` 会走 `Eclipse Curtains`。

- 这轮又继续把“换场本身怎么接管”拆成了 `Stage Takeover`：
- 它专门负责“当新舞台人格接管旧舞台时，具体是像开闸、包场、重击、扫描、冲场还是压暗”。
- 现在 `Choir Vault` 会走 `Cathedral Iris`，`Silken Halo` 会走 `Velvet Drape`，`Forge Throne` 会走 `Forge Slam`，`Phase Cloister` 会走 `Prism Scanline`，`Neon Causeway` 会走 `Runway Rush`，`Eclipse Altar` 会走 `Eclipse Veil`。
- 这层已经接入中间舞台换场过程、左上角角标和右侧 `Stage Takeover` 面板，所以现在不同 Growth 分叉连接管方式都彻底分开了。
- 这轮后端也继续把“乐理到参数”的映射深化成了系统性交互层：
- 它不再只靠固定命名组合，而是让某些已经被算出来的参数族群再进一步触发二次加成。
- 目前已经接入 `Horizon Bloom / Abyss Pressure / Slipstream Pocket / Prism Surge` 这几类 emergent synergy。
- 例如高开放度 + 高终止牵引 + 高亮度会推成 `Horizon Bloom`；高摩擦 + 高重力 + 高暗压会推成 `Abyss Pressure`；高摆动 + 高速度 + 高能量会推成 `Slipstream Pocket`；高复杂度 + 高攻击性 + 强 beam 会推成 `Prism Surge`。
- 这意味着现在后端不是只在认“你是不是命中了某条固定 combo rule”，而是已经开始识别“你堆出来的参数人格是什么”，再把它继续往更极端的舞台性格推进。
- 这轮又继续把这些 emergent synergy 放大成了 `Synergy Glyph`：
- 它专门负责“后端刚识别出来的系统性交互，在舞台上应该长成什么样的徽记结构”。
- 现在 `Horizon Bloom` 会长出日冕开叶，`Abyss Pressure` 会长出压缩环和下坠裂口，`Slipstream Pocket` 会长出追风 sweep，`Prism Surge` 会长出折线门框与 surge 骨架。
- 这层已经接入中间舞台、左上角角标和右侧 `Synergy Glyph` 面板，所以后端新长出来的参数人格，现在已经能被用户直接看到，而不是只停在数据里。
- 这轮又继续把空气介质拆成了 `Stage Climate`：
- 它专门负责“这一场到底是在暖雾里、丝绒粉雾里、热灰里、电雾里、潮汽里还是阴雾里发生”。

- 这层已经接入中间舞台、左上角角标和右侧 `Stage Climate` 面板，所以现在不同路线不仅结构不同、glyph 不同，连空气都不一样。
- 这轮又继续把“拖拽顺序”本身变成了正式的舞台参数：
- 后端 `/sandbox/render` 新增 `phrase_trajectory` 与 `phrase_trajectory_intensity`，同一组模块只要顺序不同，舞台主弧线就可能不同。
- 当前已接入的顺序轨迹包括 `Lift Arc / Velvet Drift / Forge Drop / Prism Climb / Runway Drive / Shadow Sink`。
- 这层已经接入后端引擎、前端 Canvas、左上角角标和右侧 `Phrase Trajectory` 面板，所以现在用户重新排列模块时，不再只是“数值轻微变一下”，而会明显感觉到舞台推进手势变了。
- 这轮继续把顺序再掰细一层，做成了 `Phrase Hooks`：
- 它专门负责“相邻两个模块是怎么接过去的”，不只是整句往哪走。

- 这层已经接入后端引擎、前端 Canvas、左上角 `Hooks: ...` 角标和右侧 `Phrase Hooks` 面板，所以现在用户重新排序时，不只是整体弧线会变，连局部桥接节点都会变。
- 这轮继续把 Growth 和顺序系统缝成了 `Phrase Variation`：
- 它专门负责“同一条顺序轨迹在不同成长路线下会不会被继续改写”，不再只是原轨迹上叠一层风格特效。
- 当前已接入的句法变体包括 `Choir Step / Silk Orbit / Hammer Fall / Phase Spiral / Spark Chase`。
- 这层已经接入后端 `/sandbox/render`、前端 Canvas、左上角 `Variation: ...` 角标和右侧 `Phrase Variation` 面板，所以现在同一组顺序在不同 Growth 视角下也会长成不同推进人格。

### 4. 最基础认证与用户隔离

- 已支持注册、登录、退出和会话恢复。
- 前端会把 token 保存在本地，并在刷新后恢复当前登录状态。
- 后端 `practice-records`、`heatmap`、`skill-tree`、`unlocked-effects`、`sandbox/render`、`compositions` 已全部改为按当前登录用户隔离数据。
- 不再依赖前端传 `user_id` 决定读取谁的数据，避免最基础的越权问题。
- 密码目前按项目约定保持明文方案，先满足轻量化上线和用户隔离目标。
- 后端 CORS 已支持 `localhost` 和 `IP + 端口` 形式的前端访问，便于后续直接部署到服务器用 IP 调试。

### 5. 基础上线能力

- 后端已支持直接托管 `frontend/dist`，项目可以改为单进程、单端口运行。

## 当前主要接口

- `POST /auth/register`：注册并返回 token。
- `POST /auth/login`：登录并返回 token。
- `GET /auth/me`：读取当前用户信息。
- `POST /auth/logout`：注销当前 token。
- `POST /practice-records`：创建练习记录并返回 EXP、等级、连续签到、解锁特效。
- `GET /practice-records`：读取用户最近练习记录，支持主题和日期范围筛选。
- `GET /heatmap/yearly`：读取年度练习热力图。
- `GET /skill-tree`：读取技能树进度。
- `GET /unlocked-effects`：读取用户已经永久解锁的视觉能力。
- `POST /sandbox/render`：根据乐理组合和用户解锁状态返回视觉参数。
- `POST /sandbox/render` 额外支持 `preview_growth_imprint`，用于不落库的成长风格预览。
- `POST /compositions`：保存沙盘组合。
- `GET /compositions`：读取用户保存的沙盘组合。
- `PUT /compositions/{composition_id}`：更新用户保存的沙盘组合。
- `DELETE /compositions/{composition_id}`：删除用户保存的沙盘组合。
